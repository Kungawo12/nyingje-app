### Screen-by-screen brief:

**Onboarding (3 slides):**
- Slide 1: Title "Welcome to Nyingje", subtitle "A compassionate space for your hardest moments.", large soft circle decoration in accent gold
- Slide 2: Title "Rooted in wisdom", subtitle "Drawing on Buddhist contemplative tradition and modern psychology.", simple lotus SVG or ✿ character
- Slide 3: Title "Private & safe", subtitle "Your conversations are encrypted. Only you can see them.", lock emoji or shield

Each slide: full screen, background #FDFCF8, large centered content, dot indicators at bottom.
Last slide has "Get Started" button (primary), all slides have "Skip" top-right.

**Login screen:**
- "Welcome back" heading
- Email + password inputs (surface background, border #E5E0D8, border radius 12)
- "Sign in" primary button (full width, border radius 9999)
- "Don't have an account? Register" link below
- No logo image — just "nyingje" text in large, muted style at top

**Register screen:**
- "Create your account" heading
- Name, email, password, confirm password inputs
- "Create account" primary button
- "Already have an account? Sign in" link
- Same calm style as login

**Chat screen:**
- Header: "Nyingje" centered, small subtitle "Here to listen" in muted
- Message list: FlatList, messages rendered with ChatBubble
- Input bar at bottom: ChatInput component
- Pre-populate with 2-3 mock messages to show the design:
  - AI: "Hello. I'm glad you're here. How are you feeling today?"
  - User: "I've been really anxious lately..."
  - AI: "I hear you. Anxiety can feel overwhelming. Would you like to tell me more about what's been happening?"
- AI bubble: left-aligned, background #EDE9E3, text #2C2C2C, border radius 20px (flat top-left)
- User bubble: right-aligned, background #5B7FA6, text white, border radius 20px (flat top-right)
- Keyboard avoiding view — input stays above keyboard

**ChatBubble.jsx:** accepts `{ message: string, role: 'user' | 'assistant' }`, renders accordingly

**ChatInput.jsx:** TextInput (multiline, max 4 lines) + send icon button (→). Disabled when empty.

### Technical requirements:
- Expo Router v3 file-based routing
- No external UI libraries — StyleSheet only
- SafeAreaView on every screen
- KeyboardAvoidingView on auth + chat screens
- All inputs use controlled components (useState)
- No real API calls — mock everything
- Smooth onboarding swipe using FlatList horizontal pagingEnabled
